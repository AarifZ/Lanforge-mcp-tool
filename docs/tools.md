## attenuator

### `attenuators`

List programmable RF attenuators with per-module attenuation in dB.

state 'Phantom' means the attenuator is configured but not currently
connected; only non-phantom units accept set_attenuation.

| Parameter | Type | Required | Description |
|---|---|---|---|
| system_id | string or null | no | Which system (omit when only one is configured) |

### `set_attenuation`

Set RF attenuation on an attenuator module (or all modules).

The workhorse of rate-vs-range and roaming tests: step attenuation_db
up to walk a station away from the AP. Values are dB; LANforge stores
tenths internally. Verify results with the attenuators tool.

| Parameter | Type | Required | Description |
|---|---|---|---|
| atten_id | string | yes | Attenuator entity id, e.g. '1.1.8036' (see attenuators tool) |
| attenuation_db | number | yes | Attenuation in dB (typically 0-95.5 in 0.5 dB steps) |
| module | integer | no | Module number 1-8, or 0 to set ALL modules |
| system_id | string or null | no | Which system (omit when only one is configured) |

### `run_command`

Execute ANY LANforge CLI command through the JSON API (POST /cli-json/<command>).

This is the universal write tool — anything the LANforge CLI can do is
available here. Unknown commands are forwarded to the GUI (which is the
authority), so newer LANforge features work without a server upgrade.

| Parameter | Type | Required | Description |

| confirm | boolean | no | Required true for destructive commands (rm_*, reset_*, reboot, ...) |
| system_id | string or null | no | Which system (omit when only one is configured) |

### `shell_command`

### `run_workflow`

Execute a custom workflow: any chain of commands, queries, waits,
condition polls, stat sampling, scripts, shell steps and reports.

Example — bring up traffic, watch it, report, tear down:
steps=[
  {"action":"command","command":"set_cx_state","params":{"test_mgr":"default_tm","cx_name":"udp-1","cx_state":"RUNNING"}},
  {"action":"sample","endpoint":"cx","interval_sec":5,"duration_sec":60,"register":"stats"},
  {"action":"command","command":"set_cx_state","params":{"test_mgr":"default_tm","cx_name":"udp-1","cx_state":"STOPPED"}},
  {"action":"report","title":"UDP soak","data":"${stats}"}
]

| Parameter | Type | Required | Description |
|---|---|---|---|
| steps | array | yes | Ordered steps. Each step has an 'action': command {command, params, confirm?} \| raw {line} \| query {endpoint, columns?, eids?} \| shell {shell} \| script {script, args} \| wait {seconds} \| wait_for {endpoint, eids?, until:{field,op,value,match}, timeout_sec, interval_sec} \| sample {endpoint, interval_sec, duration_sec} \| report {title, data} \| log {message}. Common fields: name, register (save result as variable), on_error (abort\|continue\|retry), retries. Strings support ${variable} substitution, including ${registered.path[0].field}. |
| variables | object | no | Initial variables for ${...} substitution |
| name | string | no | Workflow name (used in reports/logs) |
| dry_run | boolean | no | Validate and show the plan without executing |
| background | boolean | no | Return a workflow_id immediately; poll with workflow_status |
| system_id | string or null | no | Which system (omit when only one is configured) |
